# 本地化贡献指南

感谢您有兴趣为 Claude Code Router 贡献翻译！本指南将帮助您添加新语言支持或改进现有翻译。

## 🌍 当前支持的语言

- 英语 (en) - 默认
- 中文 (zh)

## 📝 如何贡献新语言

### 1. 创建语言文件

1. 进入 `src/locales/` 目录
2. 创建一个新的 JSON 文件，使用您的语言代码命名（例如：日语用 `ja.json`，西班牙语用 `es.json`）
3. 从 `en.json` 复制结构作为模板
4. 将所有字符串翻译成目标语言

### 2. 语言文件结构

```json
{
  "cli": {
    "help": {
      "usage": "您的语言中的用法说明",
      "description": "您的语言中的描述",
      "commands": {
        "title": "您的语言中的命令标题",
        // ... 更多命令描述
      }
    },
    "messages": {
      // 状态和错误消息
    }
  },
  "status": {
    // 状态显示字符串
  },
  // ... 其他部分
}
```

### 3. 注册您的语言

编辑 `src/i18n.ts` 文件以添加您的语言：

```typescript
// 将您的语言代码添加到数组中
['en', 'zh', '您的语言代码'].forEach(lang => {
  // ... 现有代码
});
```

### 4. 更新文档

1. 更新 README.md，在"语言支持"部分列出您的语言
2. 如果可能，创建您语言版本的 README（例如：`README_ja.md`）

## 🔍 翻译指南

1. **保留占位符**：不要翻译类似 `${variable}` 的变量占位符
2. **保持格式**：保留空格、换行和标点符号
3. **保持一致性**：在整个翻译中使用一致的术语
4. **考虑上下文**：某些技术术语可能不需要翻译
5. **测试您的翻译**：使用您的语言运行应用程序，确保所有内容正确显示

## 🧪 测试您的翻译

1. 构建项目：
   ```bash
   npm run build
   ```

2. 使用您的语言进行测试：
   ```bash
   export CLAUDE_ROUTER_LANG=您的语言代码
   node dist/cli.js --help
   node dist/cli.js status
   ```

3. 验证所有命令和消息是否正确显示

## 📤 提交您的翻译

1. Fork 本仓库
2. 创建新分支：`git checkout -b add-语言-translation`
3. 添加您的翻译文件
4. 提交更改：`git commit -m "Add 语言名称 translation"`
5. 推送到您的 fork：`git push origin add-语言-translation`
6. 创建 Pull Request，包含：
   - 标题："Add [语言名称] translation"
   - 描述：简要说明您的翻译
   - 截图：翻译后的界面截图（如果可能）

## 🤝 翻译审查流程

1. 母语使用者将审查翻译的准确性
2. 验证技术准确性
3. 在实际应用程序中测试翻译
4. 一旦批准，将合并并包含在下一个版本中
